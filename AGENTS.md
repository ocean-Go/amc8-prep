# AMC8 Prep Agent Instructions

## Canonical repository
All implementation work for this project must happen in:
- https://github.com/ocean-Go/amc8-prep

Do not create or use any other repository.
Do not open pull requests in any other repository.
Do not suggest moving code to another repository.

## Branching
Create a new feature branch for each Linear issue.
Branch naming format:
- feat/<short-task-name>
- fix/<short-task-name>

## Scope control
Only modify files explicitly allowed in the Linear issue.
Do not refactor unrelated modules.
Do not rename existing files unless required by the task.

## Delivery
For every task:
1. Make changes only in this repository
2. Open a PR in this repository
3. Link the PR back to the Linear issue
4. Summarize what changed, how to test, and any risks

## Product focus
This is an AMC8 training system.
Prioritize:
- real practice
- attempt tracking
- wrong-book review
- mock exam
- dashboard metrics

Do not spend time on:
- unnecessary animation
- social features
- broad refactors
- unrelated experiments
