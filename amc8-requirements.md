# AMC8 竞赛备考系统 - 需求分析文档
# ======================================
# 项目：为 Matt (11岁) 和 Chris (9岁) 准备 AMC8 竞赛
# 目标日期：2027年1月23日
# 当前日期：2026年3月2日
# 备考周期：约 10 个月

## 核心原则 🎯
**1. 优先使用 MiniMax M2.5（已付费）**
- 所有 AI 调用优先使用 MiniMax M2.5
- 仅当 MiniMax 不可用时使用 OpenAI
- Token 消耗优先级：MiniMax > OpenAI

**2. 最小化使用 Token (P0 优先级)**
- 所有静态内容不调用 API
- 题目展示、答题逻辑纯前端实现
- 仅在以下场景使用 AI：
  - 错题讲解（按需调用）
  - 个性化学习建议（每日最多1次）
  - 学习报告生成（每周最多1次）
- 缓存机制：同一问题 24 小时内不重复调用
- 预算控制：每月 Token 消耗不超过 $10

## 三、系统架构建议

### 3.1 技术栈

```
前端: React + TailwindCSS (或纯HTML/CSS/JS)
后端: Node.js + Express + SQLite/Supabase
AI: OpenAI GPT-4o (错题讲解、个性化推荐)
部署: GitHub Pages (前端) + Railway/Render (后端)
```

### 3.2 数据结构

```
用户表: id, name, age, avatar
学习记录: user_id, topic_id, status, score
错题表: user_id, question_id, error_type, times_wrong
知识点表: id, topic, subtopic, difficulty, resource_url
真题表: id, year, question, answer, explanation, topic_tags
```
