# 架构展示页优化记录（保持现有架构不变）

> 目标：在不改变“家庭 AMC8 + NotebookLM”既有分层架构与信息结构前提下，提升可维护性、可访问性与可审阅性。

## 本次优化范围

- 保持不变：
  - 核心架构层次（前端层 / 后端层 / 数据层 / 外部知识层）
  - 核心质量目标（<500ms、100% 引用率、>=80% 覆盖率、parent_id/child_id 隔离）
  - NotebookLM 作为外部知识增强定位
- 优化内容：
  1. 页面代码结构化
  2. 文档访问体验
  3. 变化点可追踪

## 变化点清单

1. `app/architecture/page.tsx`
   - 抽取 `SectionCard` 复用组件，减少重复 JSX，提高维护性。
   - 增加 `metadata`，提升页面语义与检索友好性。
   - 为分区增加 `aria-label`，优化可访问性。
   - 文档链接改为新窗口打开，避免用户离开当前总览页。
   - 新增“优化记录”链接，形成“设计文档 + 变更日志”双文档闭环。

2. `app/page.tsx`
   - 保留现有首页信息架构，仅做小幅代码清理：将静态用户数据从 `useState` 改为常量，降低无意义状态管理复杂度。

3. 文档发布路径
   - 新增 `public/docs/architecture-optimization-log.md`，确保部署后可直接访问优化记录。

## 预期收益

- **可维护性**：页面卡片结构复用，后续新增区块成本更低。
- **可审阅性**：评审可以直接看到“原方案 + 优化记录”。
- **用户体验**：查看文档时不打断当前架构总览页面。

## 兼容性说明

- 本次变更不涉及后端接口、数据库结构与权限策略。
- 不改变现有业务流程与路由拓扑，仅新增文档与页面内部组织优化。
